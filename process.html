What is happening

Sheet changes translate to the interface automatically.

When columns are removed from the workbook, the generated category cards disappear from the interface. When those columns are restored and the updater runs again, the cards come back with their project counts and links.

Step 1

The workbook structure changes first

The workbook is the source of truth for the interface. In this version, the Pipe Cleaners and 3D Pen columns are not present, so the generator only has source columns through Lego to read.

Step 2

The interface updates from those columns

After the updater reads the workbook, the interface no longer renders Pipe Cleaners or 3D Pen. Lego becomes the trailing category because it is the last category column available in the source sheet.

Step 3

The removed columns are restored

Once Pipe Cleaners and 3D Pen return to the workbook, the source again contains the full category set. No manual frontend editing is needed because the interface is rebuilt from the sheet.

Step 4

The updater regenerates the frontend data

The command passes the restored workbook into the Python generator. The script extracts categories, belts, links, resource types, and counts, then replaces the generated campData block in script.js.

Step 5

The synced interface matches the restored sheet

After the restored workbook is synced, the frontend shows the full category set again. Pipe Cleaners and 3D Pen return because those columns are back in the spreadsheet, and the generated data now contains 22 categories and 396 resources.

Script logic

How the automation works

1. Read the workbook

generateCampgrids.py opens the Excel file as a zip archive, reads the worksheet XML, shared strings, and hyperlink relationship files.

2. Build structured data

The first row becomes category cards. Belt names in column A set the current belt. Project cells become rows with title, link, and type.

3. Replace the data block

The generator leaves the rendering code alone and only rewrites the campData array in script.js.

4. Render the interface

The browser loads script.js, loops through campData, creates category cards, and nests belt subDropdowns inside each card.
